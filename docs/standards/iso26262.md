# ISO 26262 — Road Vehicle Functional Safety

go-FuSa provides automated gap assessment and evidence generation for
ISO 26262:2018 (second edition).

## Overview

ISO 26262 defines the functional safety lifecycle for electrical and electronic
(E/E) systems in road vehicles. It uses Automotive Safety Integrity Levels
(ASIL A–D, plus QM) to classify hazards and derive requirements.

## ASIL levels

| Level | Hazard probability | Controllability | Severity |
|-------|--------------------|-----------------|----------|
| QM | Quality management only | — | — |
| ASIL A | Lowest | Controllable | Low |
| ASIL B | — | — | — |
| ASIL C | — | — | — |
| ASIL D | Highest | Difficult | Severe |

## Rule mapping

| Rule ID | Clause | Description |
|---------|--------|-------------|
| `ISO26262001` | Part 6 §5.4 | Software architecture: coupling metric exceeds threshold |
| `ISO26262002` | Part 6 Table 1 | Language subset: unsafe/panic usage detected |
| `ISO26262003` | Part 8 §9.1 | Configuration management: no version tag found |

## Evidence files checked

`gofusa check --standard ISO26262` also looks for:

| File | Clause | Purpose |
|------|--------|---------|
| `hara.json` | Part 3 | Hazard Analysis and Risk Assessment |
| `fmea.json` | Part 10 | Failure Mode and Effects Analysis |
| `.fusa-reqs.json` | Part 4/6 | Derived safety requirements |

## Quick start

```sh
# Run ISO 26262 rules only
gofusa check --standard ISO26262

# Full gap report
gofusa qualify --standard ISO26262
```

## ASIL decomposition

go-FuSa supports ASIL decomposition annotations in `.fusa.yaml`:

```yaml
project:
  standard: ISO26262
  asil: D
  decomposition:
    - component: parking-brake-ctrl
      asil: B+B
```

## Part 6 software requirements

go-FuSa helps satisfy Part 6 by:

- Enforcing language subset rules (LINT001, LINT002)
- Measuring coupling between software components (COUP001–COUP003)
- Verifying requirements traceability (trace check)
- Generating Software Accomplishment Summary (SAS) evidence
