# IEC 61508 — Functional Safety of E/E/PE Systems

go-FuSa supports IEC 61508:2010 gap assessment for Safety Integrity Levels
SIL 1 through SIL 4.

## Overview

IEC 61508 is the foundational functional safety standard for electrical,
electronic and programmable electronic (E/E/PE) safety-related systems. It is
the parent standard from which ISO 26262, IEC 62443, and EN 50128 are derived.

## Safety Integrity Levels

| SIL | PFH (continuous) | PFD (demand) |
|-----|-----------------|--------------|
| SIL 1 | 10⁻⁶ – 10⁻⁵ /h | 10⁻² – 10⁻¹ |
| SIL 2 | 10⁻⁷ – 10⁻⁶ /h | 10⁻³ – 10⁻² |
| SIL 3 | 10⁻⁸ – 10⁻⁷ /h | 10⁻⁴ – 10⁻³ |
| SIL 4 | 10⁻⁹ – 10⁻⁸ /h | 10⁻⁵ – 10⁻⁴ |

## Rule mapping

| Rule ID | Clause | Description |
|---------|--------|-------------|
| `IEC61508001` | §7.4.3 | Defensive programming: panic detected |
| `IEC61508002` | §7.4.7 | Error handling: unchecked return value |
| `IEC61508003` | §7.9.2 | Modularisation: coupling exceeds threshold |

## Part 3 software requirements

IEC 61508-3 specifies software techniques recommended or mandatory at each SIL.
go-FuSa automates evidence collection for:

| Technique | SIL 1–2 | SIL 3–4 | Automated by |
|-----------|---------|---------|--------------|
| Structured programming | R | HR | LINT001, LINT002 |
| Defensive programming | R | HR | IEC61508001 |
| Error handling | R | HR | LINT003, IEC61508002 |
| Formal methods | — | R | manual |
| Independence testing | R | HR | `verify` evidence bundle |

R = Recommended, HR = Highly Recommended

## Evidence checklist

`gofusa check --standard IEC61508` verifies:

- [ ] No `panic` calls in production code (IEC61508001)
- [ ] All errors handled (IEC61508002)
- [ ] Coupling within bounds (IEC61508003)
- [ ] Requirements traced to tests (trace check)
- [ ] SBOM generated (release)

## Quick start

```sh
# Run IEC 61508 rules
gofusa check --standard IEC61508

# Generate evidence bundle
gofusa qualify --standard IEC61508 --sil 3
```
