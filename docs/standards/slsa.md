# SLSA — Supply-chain Levels for Software Artifacts

go-FuSa provides a SLSA v1.0 supply-chain integrity gap assessment to help teams achieve
and document the evidence required at each assurance level.

## Overview

[SLSA](https://slsa.dev/) is a security framework for supply-chain integrity. It defines
four assurance levels based on the strength of build provenance evidence:

| Level | Description |
|-------|-------------|
| L1 | Build provenance exists (any format) |
| L2 | Provenance generated by a hosted build platform with authenticated identity |
| L3 | Hardened build platform; provenance is non-falsifiable |
| L4 | Same scope as L3 in SLSA v1.0 (hermetic builds, two-party review) |

## Objectives assessed by `gofusa slsa`

| Level | Objective | Evidence |
|-------|-----------|----------|
| L1 | Version-controlled source (`.git` present) | `.git/` directory |
| L1 | Go module defined | `go.mod` |
| L1 | Build provenance recorded | `provenance.json` |
| L2 | Provenance includes build platform identity | `provenance.json` with `builder` field |
| L2 | Provenance includes commit reference | `provenance.json` with `vcsRevision` field |
| L2 | Software Bill of Materials present | `sbom.json` |
| L3 | Code ownership documented | `CODEOWNERS` or `.github/CODEOWNERS` |
| L3 | SBOM includes package list | `sbom.json` with non-empty `packages` array |
| L3 | Release artifact checksums present | `SHA256SUMS` or `*.sha256` file |
| L3 | Full evidence bundle produced | `audit-pack.zip` |

## Quick start

```sh
# SLSA v1.0 gap report at L2 (recommended for most CI pipelines)
gofusa slsa --level L2

# JSON output for CI gate (exits 1 when gaps exist)
gofusa slsa --level L2 --format json --output slsa-gap-report.json

# Check L3 readiness
gofusa slsa --level L3
```

## Generating the required evidence

```sh
# Generate provenance.json and sbom.json (satisfies L1/L2 provenance objectives)
gofusa release

# Add CODEOWNERS (L3) — create at repo root or under .github/
echo "* @your-org/safety-team" > CODEOWNERS

# Generate full audit bundle (L3)
gofusa audit-pack
```

## CI integration

```yaml
- run: gofusa release        # provenance.json + sbom.json
- run: gofusa slsa --level L2 --format json --output slsa-gap-report.json
- uses: actions/upload-artifact@v4
  with:
    name: slsa-evidence
    path: |
      provenance.json
      sbom.json
      slsa-gap-report.json
```

## Relationship to IEC 62443

SLSA L2/L3 evidence (provenance, SBOM, audit-pack) overlaps with IEC 62443-4-2
Component Requirements CR 3.4, CR 4.3, and CR 2.6. Running both assessments
is efficient since both read the same artifact files:

```sh
gofusa slsa --level L2
gofusa iec62443 --sl SL-2
```
