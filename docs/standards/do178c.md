# DO-178C — Software Considerations in Airborne Systems

go-FuSa supports DO-178C structural coverage analysis and mutation testing
evidence for Development Assurance Levels (DAL) A through D.

## Overview

DO-178C (with RTCA DO-178C / EUROCAE ED-12C) is the primary standard for
aviation software certification. It defines objectives for software planning,
development, verification, and quality assurance.

## Development Assurance Levels

| DAL | Effect of failure | Coverage required |
|-----|-------------------|-------------------|
| DAL-A | Catastrophic | MC/DC + decision + statement |
| DAL-B | Hazardous | Decision + statement |
| DAL-C | Major | Statement |
| DAL-D | Minor | — (recommended) |
| DAL-E | No effect | None |

## Coverage requirements

### Statement coverage (DAL-C and above)

Every executable statement must be executed by at least one test.

### Decision coverage (DAL-B and above)

Every Boolean decision must be exercised both true and false.

### MC/DC — Modified Condition/Decision Coverage (DAL-A)

Each condition in a decision must independently affect the decision outcome.

go-FuSa approximates MC/DC evidence using mutation testing:

> A mutation score ≥ 80% (all surviving mutants reviewed) provides
> MC/DC-equivalent evidence per DO-178C AC §2.3.1(b).

## Mutation testing

```sh
# Run mutation testing (requires go-mutesting in PATH)
gofusa coverage --dal A --mutate

# JSON report
gofusa coverage --dal A --mutate --format json --output mutation-report.json
```

The mutation report includes:

```json
{
  "dal": "DAL-A",
  "mutants": 142,
  "killed": 118,
  "survived": 24,
  "score": 83.1,
  "mcdcEvidence": "mutation score ≥ 80% provides MC/DC-equivalent evidence per DO-178C AC §2.3.1(b)"
}
```

## Rule mapping

|---------|---------|-----------|-------------|
| `DO178C001` | A-5 | OBJ-1 | Low-level requirements coverage gap |
| `DO178C002` | A-7 | OBJ-4 | Structural coverage below DAL threshold |
| `DO178C003` | A-7 | OBJ-5 | Decision coverage not achieved |
| `DO178C004` | A-7 | OBJ-6 | MC/DC not achieved (DAL-A) |
| `DO178C005` | A-4 | OBJ-7 | No SCI (Software Configuration Index) found |

## Quick start

```sh
# Statement coverage report for DAL-C
gofusa coverage --dal C --profile coverage.out

# Full DO-178C gap assessment
gofusa check --standard DO178C

# Cyclomatic complexity gate (§6.3.4) — threshold by DAL
gofusa comp --dal DAL-B                       # threshold 10 (DAL-B default)
gofusa comp --dal DAL-A --format json         # threshold 4, JSON output for CI

# Generate SAS (Software Accomplishment Summary)
gofusa sas --dal A
```
