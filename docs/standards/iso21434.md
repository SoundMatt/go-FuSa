# ISO 21434 — Road Vehicle Cybersecurity Engineering

go-FuSa provides automated CAL-level gap assessment and TARA evidence
checking for ISO 21434:2021.

## Overview

ISO 21434 defines cybersecurity engineering requirements for road vehicles
throughout the lifecycle: concept, product development, production, operation,
maintenance, and decommissioning.

## Cybersecurity Assurance Levels

| CAL | Risk | Example |
|-----|------|---------|
| CAL-1 | Lowest | Cosmetic/comfort feature |
| CAL-2 | Low | Infotainment system |
| CAL-3 | Medium | ADAS sensor fusion |
| CAL-4 | Highest | Brake-by-wire, powertrain |

## Running the assessment

```sh
# Assess against CAL-3 (writes gap report to stdout)
gofusa iso21434 --cal CAL-3

# JSON output
gofusa iso21434 --cal CAL-4 --format json --output iso21434-report.json

# Write to file
gofusa iso21434 --cal CAL-2 --output report.txt
```

Exit code 1 is returned when any GAP findings exist.

## Rule ISO21434001

`ISO21434001` fires at INFO level when the project standard is `ISO21434`
and `tara.json` is absent from the project root. The TARA (Threat Analysis
and Risk Assessment) is required by §8.3 for all CAL levels.

```sh
gofusa check --standard ISO21434
```

## Integration with UN R.155

ISO 21434 and UN R.155 are closely related. Use the companion command:

```sh
gofusa unece
```

to assess UN R.155 Annex 5 threat category coverage. See
`docs/standards/unece-r155.md`.
